Documentation



Commençons par le commencement. L'Ipikathon, comme son nom l'indique, est un hackathon. Il est séparé en deux parties distinctes qui sont la partie CTF qui consiste en une suite de challenges à résoudre sur différentes technologies et la partie logiciel qui comprend la conception, la réalisation et le déploiement d'une IHM sur un robot d'accueil.


La totalité des élèves est répartie de manière équitable entre 16 équipes et chaque équipe sera composée d'élèves venant des différentes filières de l'IPI ainsi que de 2 teamleaders pour favoriser la cohésion des groupes.


Vous devrez développer une interface graphique et aurez un temps d'accès au robot qui sera limité.

Partie CTF


Les Capture The Flag sont des jeux consistant à exploiter des vulnérabilités informatique de manière à récupérer des drapeaux (flag) représentant la preuve de l'intrusion.


• Vous allez devoir réaliser des challenges sur différentes technologies/langages.
• Chaque challenge rapporte un nombre de points dépendant de sa difficulté.
• Certains sont -obligatoires-. Ils sont le minimum vital à accomplir pour pouvoir accéder physiquement au robot et pouvoir tester vos interfaces mais ne seront sûrement pas suffisants pour terminer premier !
Une fois ces challenges obligatoires terminés, vous en débloquerez de nouveaux, plus complexes.
• Chaque challenge se valide par l'envoi du ou des flags qui lui correspondent.

Flags


Les Flags sont une suite de caractères pré-générés. Ils sont sous la forme : CFLAG{CCJsui1Flag} ou CFLAG{01e0b845a693187} ou Bjr_le_flag


Il s'agit là de votre nouveau but dans la vie. Ils représentent la fin d'une épreuve mais aussi sa réussite ! L'essentiel du temps, c'est leur contenu entre accolades que vous devrez rentrer sur le site pour valider le challenge. Dans certains cas, il n'y aura pas la mention CFLAG{} mais vous saurez quand même les reconnaître.





Partie logiciel


Le robot keylo est un robot pourvu de nombreux capteurs (capteurs de proximité, caméras 3D, lidar…) lui permettant de se déplacer de manière autonome dans son environnement. Il a été initialement conçu pour distribuer des cadenas (via un distributeur intégré) dans les centres de self-stockage (gardes-meubles) afin de s’affranchir de la présence d’un agent d’accueil, dans un lieu où les passages sont peu fréquents.

Actuellement répartis dans les C-zone (Comfort-zones), vous allez devoir développer une application pour d’autres cas d’usages qui vous sont proposés et qui ont été choisis par vos team-leaders.

Pour ce faire, différentes informations sont à prendre en compte pour le développement, comme les caractéristiques matérielles du robot:

  • Ecran tactile 24 pouces de résolution : 1080x1920 (portrait)
  • Un distributeur mécanique avec vis sans fin permettant de distribuer différents types d’objets
  • Une trappe sur l’avant du robot pilotée par un électro-aimant
  • Possibilité de piloter les déplacements du robot via une manette (xbox 360)
  • Une caméra frontale permettant d’enregistrer des messages vidéo / effectuer des appels vidéo / faire des photos ...
  • Le NUC (mini-ordinateur) présent sur le robot fonctionne sous Ubuntu 18.04

Stratégie

Un point essentiel et primordial à prendre en considération lors du développement est que le but premier est de développer une application visuelle permettant d’effectuer une démonstration de ce qu’il est possible de faire avec votre application – ceci sous-entends que toute action effectuée à l’écran peut ne pas avoir de d’action réelle mais tout simplement une action simulée (par exemple, l’enregistrement d’un message vidéo peut être simulé etc...).

Déploiement

Le code développé devra être placé sur le compte github qui vous a été communiqué en début de journée dans le repository "ipikathon_2018".

La racine du repository "ipikathon_2018" devra contenir un fichier "index.php" qui sera la première page de votre IHM lancée automatiquement par le robot.

Vous devrez venir voir un organisateur pour définir la plage horaire à laquelle vous souhaitez avoir accès au robot.

Pensez bien à venir réserver ce temps car ce sera sûrement la guerre des accès à la fin du hackathon.

Etant donné que vous disposez d'un temps limité, mieux vous serez organisé, plus vous augmenterez vos chances de pouvoir présenter quelque chose de construit à la fin.

Conseils & liens


  • Se rapprocher de l’ergonomie d’une application mobile en intégrant l’idée d’avoir une grande surface à l’écran.
  • S’orienter vers le développement d’une application de type SPA (Single Page Application)
  • Afin de tester votre application avant le déploiement sur le robot, penser à utiliser le mode « développeur » du navigateur en utilisant la vue adaptative (F12 puis Ctrl+Shit+M) pour avoir le même rendu que sur le robot.
  • L’utilisation de framework visuels tels que bootstrap / semantic ui … sont encouragés cependant, attention au temps passé à les « customiser » pour les rendre utilisables sur grand écran.
  • Penser avant de développer à bien effectuer des maquettes rapides à produire sur une feuille / un logiciel de dessin
  • Communiquez. N'oubliez pas que vous êtes en team, donc essayez de faire transiter au maximum les informations.
  • Profitez d'être nombreux par équipe, répartissez vous les taches pour plus d'efficacité.
  • Pensez à bien regarder partout ! Il arrive que des flags sauvages s'évanouissent dans la nature...
  • Prenez le temps de vous documenter, internet détient toutes les solutions à vos problèmes.
  • N'abandonnez pas ! Vous devez vous creuser la tête, il est normal d'avoir du mal à résoudre certains challenges !
  • Ne pas manger ni trop gras, ni trop sucré, ni trop salé. C'est mauvais pour la santé. Et la santé c'est important.
