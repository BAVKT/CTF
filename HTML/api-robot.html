Note concernant le fichier robot_id.js: Ce fichier contient les informations ci-dessous et permet de simplifier la portabilité du code sur les différents robots lors du IPIKATHON:
  • var api_key = [API_KEY]
  • var id_site = [ID_SITE]
  • var id_robot = [ID_ROBOT]
  • var host = [HOST]
  • var camera_id = [CAMERA_ID]


Initialisation de l'API

  • Javascript|
  • Paramètres
$(document).ready(function(e) {
                   wycaApi = new WycaAPI({
                      api_key,
                      id_site,
                      id_robot,
                      host,
                  });
                  wycaApi.init();
               });
Les paramètres api_key, id_site, id_robot et host de l'API sont automatiquement remplis avec les informations contenues dans le fichier robot_id.js importé en début de page.

Trappe magnétique à l'avant du robot


Events

onKeyboxIsDoorOpen
$(document).ready(function(e) {
    wycaApi = new WycaAPI({
        ...
        onKeyboxIsDoorOpen: function (is_door_open){
            is_door_open    /* true si la porte est ouverte */
        }
    });
 });
onKeyboxIsMagnetOn
$(document).ready(function(e) {
    wycaApi = new WycaAPI({
        ...
        onKeyboxIsMagnetOn: function (is_magnet_on){
            is_magnet_on    /* true si l'aimant est allumé, sinon faux(faux = la porte peut être ouverte) */
        }
    });
 });

Fonctions

KeyboxGetData
$(document).ready(function(e) {
    wycaApi = new WycaAPI({
        ...
    });
    
    /* Donne l'état de la keybox */
    wycaApi.KeyboxGetData(); /* Cet appel envoi l'évènement onKeyboxIsDoorOpen et onKeyboxIsMagnetOn */
    
 });
KeyboxSetMagnetState
$(document).ready(function(e) {
    wycaApi = new WycaAPI({
        ...
    });
    
    /* Récupérer l'état de la keybox */
    wycaApi.KeyboxSetMagnetState(magnet_on); /* magnet_on, bool, true : active l'aimant, false : désactive l'aimant */
    
 });
